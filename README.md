# panorama-mail

Email interface for Panorama agents, connecting to ProtonMail via **Proton Bridge**. Exposes email operations in three modes:

| Mode | Command | Use |
|------|---------|-----|
| **MCP** | `panorama-mail mcp` | JSON-RPC stdio server for Claude agents |
| **HTTP** | `panorama-mail serve` | REST API for other services |
| **CLI** | `panorama-mail <command>` | Direct command-line use |

Fetched emails are automatically persisted to a local SQLite database with Tantivy full-text indexing at `STORE_PATH` (defaults to `~/.panorama`).

---

## Requirements

- **Rust 1.94+** (edition 2024)
- **Proton Bridge** running and unlocked on the local machine
- **ProtonMail account** with Bridge configured

## Configuration

Copy `.env.example` to `.env` and fill in your values:

```bash
cp .env.example .env
```

| Variable | Default | Required | Description |
|----------|---------|----------|-------------|
| `PROTON_USERNAME` | — | **yes** | Your ProtonMail address (also used as the `From:` address) |
| `PROTON_PASSWORD` | — | **yes** | **Bridge-specific password** shown in Bridge → "Configure your email client" — not your account password |
| `IMAP_HOST` | `127.0.0.1` | no | Proton Bridge IMAP host |
| `IMAP_PORT` | `1143` | no | Proton Bridge IMAP port |
| `SMTP_HOST` | `127.0.0.1` | no | Proton Bridge SMTP host |
| `SMTP_PORT` | `1025` | no | Proton Bridge SMTP port |
| `MAILBOX` | `INBOX` | no | Mailbox to read from |
| `HTTP_PORT` | `3500` | no | Port for the HTTP REST server |
| `STORE_PATH` | `~/.panorama` | no | Base directory for the local SQLite + Tantivy store |

> **Note:** `PROTON_PASSWORD` is the password generated by Bridge itself, not your ProtonMail login password. You can find it in Bridge under **Settings → Configure your email client**.

## Starting Proton Bridge

Proton Bridge must be running before starting panorama-mail. Install it from [proton.me/mail/bridge](https://proton.me/mail/bridge).

```bash
# Start Bridge (GUI or headless)
proton-bridge --no-window       # headless mode (Linux/macOS)
```

Bridge listens on:
- IMAP: `127.0.0.1:1143`
- SMTP: `127.0.0.1:1025`

Verify Bridge is running before starting the server:

```bash
nc -zv 127.0.0.1 1143 && echo "IMAP OK"
nc -zv 127.0.0.1 1025 && echo "SMTP OK"
```

## Usage

### MCP Server (for Claude agents)

Starts a JSON-RPC 2.0 server on **stdin/stdout** that Claude can use as an MCP tool server.

```bash
panorama-mail mcp
```

Add to your Claude MCP configuration:

```json
{
  "mcpServers": {
    "panorama-mail": {
      "command": "/path/to/panorama-mail",
      "args": ["mcp"]
    }
  }
}
```

**Available MCP tools:**

| Tool | Description | Required args |
|------|-------------|---------------|
| `mail_send` | Send an email via Proton Bridge | `to`, `subject`, `body` |
| `mail_fetch_unread` | Fetch all unread messages | — |
| `mail_search` | Search by subject or sender | `query` |
| `mail_get_by_uid` | Fetch a single message by UID | `uid` |
| `mail_mark_read` | Mark a message as read | `uid` |
| `mail_list_mailboxes` | List all IMAP folders | — |

---

### HTTP Server

Starts an Axum REST API.

```bash
panorama-mail serve
# Listening on 0.0.0.0:3500 (or HTTP_PORT)
```

**Endpoints:**

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/health` | Health check → `{"status":"ok","version":"..."}` |
| `POST` | `/mail/send` | Send email — body: `{"to":"...","subject":"...","body":"..."}` |
| `GET` | `/mail/unread` | Fetch unread messages → `[AgentMessage]` |
| `GET` | `/mail/search?q=<term>` | Search messages → `[AgentMessage]` |
| `GET` | `/mail/messages/{uid}` | Fetch one message by IMAP UID → `AgentMessage` |
| `POST` | `/mail/messages/{uid}/read` | Mark message as read → `{"ok":true}` |
| `GET` | `/mail/mailboxes` | List mailbox names → `["INBOX","Sent",...]` |

**Example:**

```bash
# Send an email
curl -s -X POST http://localhost:3500/mail/send \
  -H 'Content-Type: application/json' \
  -d '{"to":"someone@example.com","subject":"Hello","body":"Test message"}'

# Fetch unread
curl -s http://localhost:3500/mail/unread | jq .

# Search
curl -s "http://localhost:3500/mail/search?q=invoice" | jq .
```

---

### CLI

Direct command-line operations. All commands load config from the environment / `.env`.

```bash
# Send an email
panorama-mail send --to someone@example.com --subject "Hello" --body "Hi there"

# Fetch unread messages (pretty-printed)
panorama-mail fetch

# Fetch unread as JSON
panorama-mail fetch --json

# Search messages
panorama-mail search "invoice"
panorama-mail search "invoice" --json

# Get a specific message by IMAP UID
panorama-mail get 42
panorama-mail get 42 --json

# Mark a message as read
panorama-mail read 42

# List all mailboxes
panorama-mail mailboxes
panorama-mail mailboxes --json
```

## Message Format

All endpoints and MCP tools return `AgentMessage` objects:

```json
{
  "uid": 42,
  "subject": "Hello from Panorama",
  "from": "sender@example.com",
  "to": ["recipient@example.com"],
  "date": "2026-05-08T14:30:00",
  "body": "Plain text body...",
  "html_body": "<p>HTML body...</p>",
  "attachments": []
}
```

## Local Store

Every email fetched from IMAP is automatically saved to a local store at `STORE_PATH` (default `~/.panorama`):

```
~/.panorama/
├── panorama.db       # SQLite — structured email storage
└── tantivy/          # Tantivy full-text search index
```

The store enables:
- **Offline access** to previously fetched messages
- **Full-text search** across subject, sender, recipients, body, and date
- **Read-state tracking** independent of IMAP (upserts preserve the `is_read` flag)

The store is designed to support multiple backends in the future via the `EmailStore` trait in `src/store/database.rs`.

## Troubleshooting

**`IMAP error: No: [AUTHENTICATIONFAILED]`**
→ Check `PROTON_PASSWORD` is the Bridge password, not your account password. Re-copy it from Bridge.

**Bridge not reachable on port 1143/1025**
→ Ensure Bridge is running and your account is unlocked in the Bridge UI.

**Store errors on first run**
→ The store directory is created automatically. If you see permission errors, check that `STORE_PATH` is writable.

**Tests skip get/mark-read steps**
→ Email delivery via Proton Bridge can take 3–10 seconds. The test suite waits 3 seconds; slow Bridge environments may need more time. This is expected and does not indicate a bug.
